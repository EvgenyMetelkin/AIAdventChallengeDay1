Ниже — оптимальное решение задачи **Two Sum** на Python с временной сложностью **O(n)**.

> **Важно:** в приведённом тобой примере есть несоответствие:  
> `nums = [2, 7, 11, 15], target = 19`  
> Пара `[0, 1]` не подходит, потому что `2 + 7 = 9`, а не `19`.  
> Для `target = 19` корректный ответ для этого массива вообще отсутствует.  
> Поэтому в примере использования я покажу **корректный тест**.

---

## Решение

```python
from typing import List

def two_sum(nums: List[int], target: int) -> List[int]:
    # Словарь: значение числа -> его индекс
    seen = {}

    for i, num in enumerate(nums):
        # Число, которое нужно найти, чтобы сумма дала target
        complement = target - num

        # Если нужное число уже встречалось, нашли ответ
        if complement in seen:
            return [seen[complement], i]

        # Сохраняем текущее число и его индекс
        seen[num] = i

    # По условию решение всегда существует,
    # но на случай некорректного ввода:
    raise ValueError("Решение не найдено")
```

---

## Как работает алгоритм

Идея такая:

- Проходим по массиву **один раз**
- Для каждого числа `num` считаем, какое число нужно до `target`:
  `complement = target - num`
- Проверяем, встречалось ли это число раньше
- Если да — возвращаем индексы
- Если нет — сохраняем текущее число в словарь

### Пример
Пусть:

```python
nums = [2, 7, 11, 15]
target = 9
```

Ход работы:

1. `i = 0`, `num = 2`  
   `complement = 7`  
   `7` ещё не встречалось → сохраняем `2: 0`

2. `i = 1`, `num = 7`  
   `complement = 2`  
   `2` уже есть в словаре → ответ `[0, 1]`

---

## Почему сложность O(n)

Мы проходим по массиву ровно **один раз**, то есть выполняем `n` итераций.

На каждой итерации:
- вычисляем `complement`
- проверяем наличие в словаре
- при необходимости добавляем элемент в словарь

Операции поиска и вставки в словарь в Python в среднем выполняются за **O(1)**.

Поэтому общая временная сложность:

\[
O(n) \cdot O(1) = O(n)
\]

---

## Анализ сложности

### Временная сложность
- **O(n)** — один проход по массиву

### Пространственная сложность
- **O(n)** — в худшем случае мы сохраним в словарь почти все элементы массива

## Пример использования

### Корректный тестовый случай
```python
nums = [2, 7, 11, 15]
target = 9

result = two_sum(nums, target)
print(result)  # [0, 1]
```

### Ещё один пример
```python
nums = [3, 2, 4]
target = 6

result = two_sum(nums, target)

```
