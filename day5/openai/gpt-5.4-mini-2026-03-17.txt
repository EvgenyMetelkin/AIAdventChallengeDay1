Вот оптимальное решение задачи **Two Sum** на Python за **O(n)**.

## Идея
Мы проходим по массиву один раз и храним уже встреченные числа в словаре `seen`:
- ключ — число из массива
- значение — его индекс

Для каждого `num` проверяем, существует ли число `target - num` среди уже просмотренных.  
Если существует — мы нашли ответ.

---

## Код

```python
def two_sum(nums, target):
    # Словарь: число -> его индекс
    seen = {}

    # Проходим по массиву один раз
    for i, num in enumerate(nums):
        complement = target - num  # Число, которое нужно найти в паре

        # Если complement уже встречался, возвращаем индексы
        if complement in seen:
            return [seen[complement], i]

        # Иначе запоминаем текущее число и его индекс
        seen[num] = i

    # По условию задачи решение всегда существует
    return []


# Пример использования
nums = [2, 7, 11, 15]
target = 9

result = two_sum(nums, target)
print(result)  # [0, 1]
```

---

## Почему алгоритм работает за O(n)

- Мы проходим массив **ровно один раз**.
- На каждой итерации выполняем:
  - вычисление `complement`
  - поиск в словаре `complement in seen`
  - добавление элемента в словарь

Все операции со словарём в среднем выполняются за **O(1)**.

Значит:
- один проход по `n` элементам,
- каждое действие на шаге — O(1),

итого: **O(n)**.

---

## Анализ сложности

### Временная сложность
- **O(n)** — один проход по массиву.

### Пространственная сложность
- **O(n)** — в худшем случае словарь `seen` может сохранить почти все элементы массива.
