```python
def two_sum(nums, target):
    """
    Two Sum (оптимальное решение за O(n))
    
    Возвращает индексы (i, j) такие что nums[i] + nums[j] == target.
    Предполагается, что ровно одно решение существует.
    Нельзя использовать один и тот же элемент дважды.
    """
    # словарь: значение числа -> его индекс
    seen = {}

    # идём по массиву один раз
    for i, x in enumerate(nums):
        # какое число нужно, чтобы вместе с x получить target
        need = target - x

        # если такое число уже встречалось, то нашли пару
        if need in seen:
            return [seen[need], i]

        # иначе запоминаем текущий элемент
        seen[x] = i


# ---- пример использования ----
if __name__ == "__main__":
    nums = [2, 7, 11, 15]
    target = 19
    print(two_sum(nums, target))  # ожидаемый вывод: [0, 1]
```

### Почему алгоритм имеет сложность **O(n)**
- Мы выполняем один проход по массиву: `for i, x in enumerate(nums)` — это даёт **n итераций**.
- На каждой итерации:
  - вычисляем `need = target - x` — это **O(1)**;
  - проверяем `if need in seen` — в среднем для словаря Python это **O(1)** (хеш-таблица);
  - выполняем вставку `seen[x] = i` — также **O(1)** в среднем.
- Итого: **O(n)** в среднем.

### Сложность
- **Временная:** **O(n)** (один проход, операции со словарём в среднем O(1)).
- **Пространственная:** **O(n)** в худшем случае, потому что мы можем сохранить почти все элементы массива в словаре.

### Тестовый пример
- Вход: `nums = [2, 7, 11, 15]`, `target = 19`
- Находим: `2 + 7 = 19`
- Выход: `[0, 1]` ✅
