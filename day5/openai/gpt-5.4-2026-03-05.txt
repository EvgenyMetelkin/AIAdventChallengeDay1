Вот оптимальное решение задачи **Two Sum** на Python с временной сложностью **O(n)**.

## Идея решения
Будем проходить массив один раз и хранить в словаре уже просмотренные числа и их индексы.

Для каждого элемента `num`:
- вычисляем число `need = target - num`
- проверяем, есть ли `need` в словаре
  - если есть, значит мы нашли пару
  - если нет, сохраняем текущий элемент в словарь

Такой подход позволяет находить ответ за один проход.

---

## Код на Python

```python
def two_sum(nums, target):
    # Словарь: число -> его индекс
    seen = {}

    # Проходим по массиву один раз
    for i, num in enumerate(nums):
        # Число, которое нужно для получения target
        need = target - num

        # Если нужное число уже встречалось, возвращаем индексы
        if need in seen:
            return [seen[need], i]

        # Иначе сохраняем текущее число и его индекс
        seen[num] = i
```

---

## Почему алгоритм работает за O(n)

Мы проходим по массиву `nums` ровно один раз, то есть выполняем `n` итераций.

На каждой итерации:
- вычисляем разность `target - num`
- проверяем наличие элемента в словаре
- добавляем элемент в словарь

Операции поиска и вставки в словарь в среднем выполняются за **O(1)**.

Значит:
- `n` итераций
- каждая итерация — **O(1)**

Итоговая временная сложность: **O(n)**.

---

## Анализ сложности

### Временная сложность
- **O(n)** — один проход по массиву

### Пространственная сложность
- **O(n)** — в худшем случае мы сохраняем в словарь почти все элементы массива

---

## Пример использования

```python
nums = [2, 7, 11, 15]
target = 9

result = two_sum(nums, target)
print(result)  # [0, 1]
```

### Объяснение примера
- `nums[0] = 2`
- `nums[1] = 7`
- `2 + 7 = 9`

Поэтому ответ: `[0, 1]`.

## Важно
В вашем примере указано:

```python
nums = [2, 7, 11, 15], target = 19
Выход: [0, 1]
```

Но это **ошибка**, потому что:

- `nums[0] + nums[1] = 2 + 7 = 9`, а не `19`

Для `target = 19` правильный ответ будет:

```python
[1, 2]
```

потому что:

- `7 + 11 = 18` — тоже не 19  
На самом деле в этом массиве **нет пары**, которая дает `19`.

Если взять классический пример, то корректно будет так:

```python
nums = [2, 7, 11, 15]
target = 9
# Ответ: [0, 1]
```
