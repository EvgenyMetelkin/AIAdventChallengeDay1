### Оптимальное решение с использованием словаря (хэш-таблицы)

Это решение использует словарь для хранения уже пройденных чисел и их индексов. Это позволяет нам найти нужную пару за один проход по массиву.

```python
from typing import List

def two_sum(nums: List[int], target: int) -> List[int]:
    """
    Находит индексы двух чисел в списке, сумма которых равна target.

    Args:
        nums: Список целых чисел.
        target: Целевая сумма.

    Returns:
        Список из двух индексов [i, j], таких что nums[i] + nums[j] == target.
    """
    # Создаем словарь для хранения чисел, которые мы уже видели,
    # и их индексов. Ключ - число, значение - индекс.
    seen_map = {}

    # Проходим по списку nums, получая доступ к индексу и значению каждого элемента.
    for i, num in enumerate(nums):
        # Для текущего числа `num` вычисляем "дополнение" (complement),
        # которое нужно найти в массиве, чтобы в сумме получить target.
        complement = target - num

        # Проверяем, есть ли это дополнение в нашем словаре.
        # Если есть, значит, мы нашли нужную пару.
        if complement in seen_map:
            # Возвращаем индекс дополнения (который мы сохранили ранее)
            # и индекс текущего числа.
            return [seen_map[complement], i]

        # Если дополнение не найдено, добавляем текущее число и его индекс
        # в словарь для будущих проверок. Мы делаем это в конце,
        # чтобы не использовать один и тот же элемент дважды.
        seen_map[num] = i

    # По условию задачи решение всегда существует, поэтому эта часть кода
    # никогда не будет достигнута. Но для полноты можно вернуть пустой список
    # или вызвать исключение, если решения может не быть.
    return []

```

### Объяснение алгоритма и его сложности

#### 1. Почему алгоритм имеет сложность O(n)?

Алгоритм достигает временной сложности **O(n)**, потому что мы проходим по входному массиву `nums` **ровно один раз**.

- Мы используем цикл `for`, который итерируется по каждому из `n` элементов массива.
- Внутри цикла все операции выполняются за **константное время O(1)** (в среднем):
    - **Вычисление `complement`**: Это простая арифметическая операция.
    - **Проверка `complement in seen_map`**: Поиск ключа в словаре (хэш-таблице) в среднем занимает O(1).
    - **Добавление `seen_map[num] = i`**: Добавление элемента в словарь также в среднем занимает O(1).

Поскольку мы выполняем `n` итераций, и каждая итерация стоит O(1), общая временная сложность алгоритма составляет **O(n) * O(1) = O(n)**.

#### 2. Анализ временной и пространственной сложности

*   **Временная сложность (Time Complexity): O(n)**
    Как объяснено выше, мы делаем один проход по массиву длиной `n`, и операции внутри цикла выполняются за константное время.

*   **Пространственная сложность (Space Complexity): O(n)**
    Мы используем дополнительную структуру данных — словарь `seen_map`. В худшем случае, если искомая пара чисел находится в конце массива, нам придется сохранить почти все `n` элементов в словаре. Таким образом, объем памяти, занимаемый словарем, растет линейно с размером входного массива, что дает нам пространственную сложность **O(n)**.

### 3. Пример использования

Вот как можно использовать эту функцию с тестовым случаем из условия задачи.

```python
# Тестовый случай
nums_example = [2, 7, 11, 15]
target_example = 19

# Вызов функции
result = two_sum(nums_example, target_example)

# Вывод результата
print(f"Входной массив: {nums_example}")
print(f"Целевая сумма: {target_example}")
print(f"Индексы искомых чисел: {result}")

# Проверка
if result:
    idx1, idx2 = result
    print(f"Проверка: {nums_example[idx1]} + {nums_example[idx2]} = {nums_example[idx1] + nums_example[idx2]}")

print("-" * 20)

# Другой пример
nums_example_2 = [3, 2, 4]
target_example_2 = 6
result_2 = two_sum(nums_example_2, target_example_2)
print(f"Входной массив: {nums_example_2}")
print(f"Целевая сумма: {target_example_2}")
print(f"Индексы искомых чисел: {result_2}") # Ожидаемый вывод: [1, 2]
```

Индексы искомых чисел: [0, 1]

Давайте исправим целевое значение в примере для наглядности. Если `target = 9`, то `2 + 7` подходит. Если `target = 26`, то `11+15` подходит. Возьмем `target = 9` для первого примера.

```python
# Исправленный пример

print(f"Индексы искомых чисел: {result}")
if result:
    idx1, idx2 = result
    print(f"Проверка: {nums_example[idx1]} + {nums_example[idx2]} = {nums_example[idx1] + nums_example[idx2]}")
```

**Правильный вывод:**
