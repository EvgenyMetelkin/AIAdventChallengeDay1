Для решения задачи за один проход с оптимальной сложностью используется **хеш-таблица** (в Python — словарь `dict`).

### Решение на Python

```python
def two_sum(nums, target):
    # Создаем словарь для хранения значений и их индексов: {значение: индекс}
    prev_map = {}
    
    for i, num in enumerate(nums):
        # Вычисляем число, которое нам нужно найти, чтобы получить target
        diff = target - num
        
        # Если это число уже есть в словаре, значит мы нашли пару
        if diff in prev_map:
            return [prev_map[diff], i]
        
        # Иначе добавляем текущее число и его индекс в словарь
        prev_map[num] = i

# Тестовый случай
nums = [2, 7, 11, 15]
target = 9 # Исправил target на 9, чтобы получить индексы [0, 1] (2+7=9)
print(f"Индексы: {two_sum(nums, target)}")
```

---

### Анализ сложности

#### 1. Временная сложность: O(n)
Мы проходим по массиву `nums` ровно один раз (один цикл `for`). Внутри цикла мы выполняем операции поиска в словаре (`diff in prev_map`) и добавления в словарь (`prev_map[num] = i`). В Python среднее время выполнения этих операций составляет **O(1)**. Таким образом, общее время работы алгоритма пропорционально количеству элементов в массиве, то есть **O(n)**.

#### 2. Пространственная сложность: O(n)
В худшем случае (если пара находится в самом конце или отсутствует) нам придется сохранить в словаре почти все элементы массива. Следовательно, количество записей в словаре может достигать `n`, что требует **O(n)** дополнительной памяти.

---

### Почему это оптимально?
*   **Наивный подход** (перебор всех пар двумя вложенными циклами) имел бы сложность **O(n²)**, что очень медленно для больших массивов.
*   **Подход с сортировкой** (указатели с двух сторон) работал бы за **O(n log n)**, что быстрее, чем наивный подход, но всё равно проигрывает решению с хеш-таблицей.
*   **Данный алгоритм (хеш-таблица)** является максимально быстрым, так как мы жертвуем памятью (O(n)), чтобы получить линейное время выполнения (O(n)).

*Примечание к вашему примеру: в условии вы указали `target = 19` для массива `[2, 7, 11, 15]`. Сумма 2+7=9, а не 19. Если вам нужно получить 19, искомая пара была бы `[1, 2]` (7+11=18 — всё еще не 19) или `[0, 3]` (2+15=17). Для 19 корректной пары в данном массиве нет.*
