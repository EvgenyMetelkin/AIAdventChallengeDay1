## Оптимальное решение задачи Two Sum на Python

### Условие задачи:

Дан массив целых чисел `nums` и целевое число `target`. Найди два индекса `i` и `j`, таких что `nums[i] + nums[j] == target`. Предположи, что каждый вход имеет ровно одно решение, и ты не можешь использовать один и тот же элемент дважды.

### Требования:

1.  Напиши решение с временной сложностью O(n).
2.  Объясни, почему твой алгоритм имеет сложность O(n).
3.  Проанализируй временную и пространственную сложность.
4.  Добавь комментарии к коду.
5.  Приведи пример использования с тестовым случаем.

---

### Решение:

```python
def twoSum(nums: list[int], target: int) -> list[int]:
    """
    Находит два индекса в массиве nums, сумма элементов которых равна target.

    Args:
        nums: Список целых чисел.
        target: Целевое число.

    Returns:
        Список из двух индексов i и j, таких что nums[i] + nums[j] == target.
        Предполагается, что ровно одно решение существует.
    """
    # Создаем словарь для хранения чисел и их индексов.
    # Ключ - число, значение - его индекс.
    num_map = {}

    # Проходим по каждому числу в массиве nums вместе с его индексом.
    for i, num in enumerate(nums):
        # Вычисляем "дополнительное" число, которое нам нужно найти,
        # чтобы в сумме с текущим числом получить target.
        complement = target - num

        # Проверяем, есть ли "дополнительное" число в нашем словаре.
        if complement in num_map:
            # Если "дополнительное" число найдено в словаре,
            # это означает, что мы нашли пару.
            # Возвращаем индекс "дополнительного" числа (из словаря)
            # и текущий индекс i.
            return [num_map[complement], i]

        # Если "дополнительное" число не найдено, добавляем текущее число
        # и его индекс в словарь. Это пригодится для будущих итераций,
        # когда текущее число может стать "дополнительным" для следующего.
        num_map[num] = i

    # В соответствии с условием задачи, ровно одно решение существует,
    # поэтому этот код никогда не будет достигнут.
    # Однако, для полноты, можно вернуть пустой список или вызвать исключение.
    return []

```

### Объяснение алгоритма и временной сложности O(n):

Алгоритм использует **словарь (хеш-таблицу)** для эффективного поиска. Идея заключается в следующем:

1.  Мы итерируемся по массиву `nums` один раз.
2.  Для каждого числа `num` в массиве, мы вычисляем `complement` - число, которое нам нужно найти, чтобы в сумме с `num` получить `target` (`complement = target - num`).
3.  Затем мы проверяем, существует ли `complement` уже в нашем словаре `num_map`.
    *   **Если `complement` есть в словаре**: Это значит, что мы уже встречали число, которое является дополнением к текущему `num` для достижения `target`. Мы нашли решение! Словарь хранит индекс ранее встреченного `complement`. Мы возвращаем этот сохраненный индекс и текущий индекс `i`.
    *   **Если `complement` нет в словаре**: Это значит, что текущее число `num` не является дополнением к каким-либо ранее встреченным числам. В этом случае мы добавляем текущее число `num` и его индекс `i` в словарь `num_map`. Это делается для того, чтобы `num` могло стать "дополнением" для чисел, которые мы встретим позже в массиве.

**Почему временная сложность O(n)?**

*   Мы проходим по массиву `nums` **ровно один раз**. Эта итерация занимает O(n) времени, где `n` - количество элементов в `nums`.
*   Операции со словарем (вставка `num_map[num] = i` и проверка наличия `complement in num_map`) в среднем выполняются за **O(1)** (константное время). Это ключевое преимущество хеш-таблиц. В худшем случае (очень редкие коллизии) они могут занимать O(n), но на практике для большинства реализаций хеш-таблиц средняя производительность именно O(1).
*   Поскольку основная работа выполняется за одну итерацию по массиву, где каждая операция внутри цикла занимает в среднем константное время, общая временная сложность алгоритма составляет **O(n)**.

---

### Анализ временной и пространственной сложности:

*   **Временная сложность:** **O(n)**
    *   Как объяснено выше, мы проходим по массиву один раз, и операции со словарем в среднем занимают O(1).

*   **Пространственная сложность:** **O(n)**
    *   В худшем случае, нам может потребоваться сохранить все элементы массива в словаре `num_map`, если "идеальная" пара находится в самом конце. Например, если `nums = [1, 2, 3, 4]` и `target = 7`, мы добавим `1, 2, 3` в словарь перед тем, как найти `4` и `3` (который уже есть в словаре). Следовательно, размер словаря может достигать `n`.
    *   Таким образом, для хранения данных в словаре нам потребуется дополнительная память, пропорциональная размеру входного массива, что дает нам пространственную сложность **O(n)**.

### Правильный пример использования с тестовым случаем:

**Вход:**
`nums = [2, 7, 11, 15]`
`target = 9`

**Вызов функции:**
```python
result = twoSum(nums, target)
print(result)
```

**Ожидаемый вывод:**
`[0, 1]`

**Пошаговое выполнение для данного примера:**

1.  `num_map = {}`
2.  **Итерация 1:** `i = 0`, `num = 2`.
    *   `complement = 9 - 2 = 7`.
    *   `7` нет в `num_map`.
    *   `num_map = {2: 0}`.
3.  **Итерация 2:** `i = 1`, `num = 7`.
    *   `complement = 9 - 7 = 2`.
    *   `2` **есть** в `num_map`!
    *   Возвращаем `[num_map[2], i]`, что равно `[0, 1]`.

Алгоритм успешно нашел индексы `0` и `1`, потому что `nums[0] + nums[1] = 2 + 7 = 9`.
