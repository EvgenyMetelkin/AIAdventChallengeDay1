Задача Two Sum является классической проблемой в информатике, которая хорошо демонстрирует преимущества использования хеш-таблиц (словари в Python) для оптимизации поиска.

### Оптимальное решение задачи Two Sum на Python

```python
def two_sum(nums: list[int], target: int) -> list[int]:
    """
    Находит два индекса элементов в массиве nums, сумма которых равна target.

    Args:
        nums: Список целых чисел.
        target: Целевое число, которое должна составить сумма двух элементов.

    Returns:
        Список из двух индексов [i, j], таких что nums[i] + nums[j] == target.
        Предполагается, что всегда существует ровно одно решение.
    """
    # Создаем словарь для хранения уже пройденных чисел и их индексов.
    # Ключ: число, Значение: его индекс в исходном массиве nums.
    # Это позволяет нам быстро проверять, видели ли мы 'дополнение' к текущему числу.
    num_map = {}

    # Итерируем по массиву nums, сохраняя как индекс, так и значение.
    for i, num in enumerate(nums):
        # Для текущего числа `num` мы ищем "дополнение" (complement),
        # которое равно `target - num`.
        # Если `complement` существует в массиве, то `num + complement == target`.
        complement = target - num

        # Проверяем, есть ли `complement` в нашем словаре `num_map`.
        # Поиск в словаре (хеш-таблице) имеет среднюю временную сложность O(1).
        if complement in num_map:
            # Если `complement` найден, это означает, что мы уже видели число,
            # которое в сумме с текущим `num` дает `target`.
            # Возвращаем индекс `complement` (который хранится в `num_map`)
            # и текущий индекс `i` числа `num`.
            return [num_map[complement], i]

        # Если `complement` не найден, добавляем текущее число `num` и его индекс `i`
        # в наш словарь. Это число может быть `complement` для какого-то будущего числа.
        # Вставка в словарь (хеш-таблицу) также имеет среднюю временную сложность O(1).
        num_map[num] = i

    # Согласно условию задачи, всегда существует ровно одно решение.
    # Поэтому эта часть кода не должна быть достигнута при корректных входных данных.
    return []

```

### 1. Объяснение временной сложности O(n)

Алгоритм имеет временную сложность **O(n)**, потому что мы проходим по входному массиву `nums` ровно один раз. В каждой итерации цикла выполняются следующие операции:
*   Вычисление `complement = target - num`: это операция с постоянным временем **O(1)**.
*   Проверка `if complement in num_map`: поиск элемента в хеш-таблице (словаре Python) имеет среднюю временную сложность **O(1)**. В худшем случае (при очень неудачных хеш-коллизиях) это может быть O(n), но на практике и для большинства случаев это O(1).
*   Добавление элемента в `num_map` (`num_map[num] = i`): вставка элемента в хеш-таблицу также имеет среднюю временную сложность **O(1)**.

Поскольку все операции внутри цикла выполняются за константное время **O(1)**, и сам цикл выполняется `n` раз (где `n` - длина массива `nums`), общая временная сложность алгоритма составляет **O(n)**.

### 2. Анализ временной и пространственной сложности

*   **Временная сложность (Time Complexity): O(n)**
    Как объяснено выше, алгоритм делает один проход по массиву, и каждая операция внутри цикла занимает в среднем постоянное время.

*   **Пространственная сложность (Space Complexity): O(n)**
    В худшем случае (например, если решение находится на последней итерации или если `complement` никогда не встречается до конца, чего не может быть по условию задачи), словарь `num_map` может хранить до `n-1` элементов. Каждый элемент в словаре занимает константное количество памяти. Таким образом, объем памяти, используемый словарем, прямо пропорционален количеству элементов в исходном массиве.
