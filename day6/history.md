### Краткое описание принятых решений

1. **Разделение ответственности**: `Agent` не знает о CLI (нет `print`, `argparse`), `CLI` не знает о деталях API.
2. **Асинхронность** выбрана для возможности расширения (стриминг, параллельные запросы) и сохранения отзывчивости.
3. **OpenAI‑совместимый формат** позволяет использовать любые LLM (OpenAI, локальные vLLM, Ollama через прокси).
4. **История диалога** – основа контекстных разговоров; `reset` и сохранение в JSON дают гибкость.
5. **Обработка ошибок** – понятные сообщения для всех типов сбоев (ключ, сеть, API).
6. **Конфигурация** через `.env` + аргументы CLI – стандартный и удобный способ.
7. **Тесты** – изоляция агента через `unittest.mock`, покрытие основного сценария и ошибок.
