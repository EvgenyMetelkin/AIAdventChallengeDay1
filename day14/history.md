### Краткое описание принятых решений

1. **Разделение ответственности**: `Agent` не знает о CLI (нет `print`, `argparse`), `CLI` не знает о деталях API.
2. **Асинхронность** выбрана для возможности расширения (стриминг, параллельные запросы) и сохранения отзывчивости.
3. **OpenAI‑совместимый формат** позволяет использовать любые LLM (OpenAI, локальные vLLM, Ollama через прокси).
4. **История диалога** – основа контекстных разговоров; `reset` и сохранение в JSON дают гибкость.
5. **Обработка ошибок** – понятные сообщения для всех типов сбоев (ключ, сеть, API).
6. **Конфигурация** через `.env` + аргументы CLI – стандартный и удобный способ.
7. **Тесты** – изоляция агента через `unittest.mock`, покрытие основного сценария и ошибок.


### 4. Объяснение изоляции сессий

Каждый клиент получает `session_id` через `Set-Cookie: session_id=...`. Этот идентификатор хранится в браузере и автоматически прикладывается к последующим запросам. Сервер использует глобальный словарь `sessions: Dict[str, Agent]`, где ключ – `session_id`, значение – экземпляр `Agent`. У каждого агента своя собственная `conversation_history`. Благодаря этому:

- Диалоги разных пользователей (или даже разных вкладок одного пользователя, если они используют разные сессии) полностью изолированы.
- При сбросе истории (`/reset`) вызывается `agent.reset_conversation()`, но сам агент остаётся в словаре (история очищается).
- При перезапуске сервера все сессии теряются – это ожидаемое поведение для простого in‑memory решения. При желании можно добавить сохранение в файлы по аналогии с `agent.save_history()`.

### 5. Пример запуска и тестирования

1. Создайте файл `.env` с вашим ключом API.
2. Установите зависимости.
3. Запустите `python web_app.py`.
4. Откройте браузер на `http://localhost:8000`.
5. Напечатайте сообщение – получите ответ от LLM.
6. Откройте другую вкладку в режиме инкогнито – там начнётся отдельный диалог.
7. Нажмите "Reset conversation" – история очистится.

**Проверка через терминал (curl)**:

```bash
# Первый запрос (создаст сессию, сохраните cookie в файл)
curl -c cookies.txt -X POST http://localhost:8000/chat \
  -H "Content-Type: application/json" \
  -d '{"message":"Hi"}'

# Следующий запрос с той же сессией
curl -b cookies.txt -X POST http://localhost:8000/chat \
  -H "Content-Type: application/json" \
  -d '{"message":"What was my first question?"}'

# Сброс
curl -b cookies.txt -X POST http://localhost:8000/reset
