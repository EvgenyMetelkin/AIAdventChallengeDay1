# Класс `Agent`

Класс для асинхронного взаимодействия с большими языковыми моделями (LLM) через OpenAI-совместимый API. Обеспечивает управление историей диалога, сохранение/загрузку контекста на диск и поддержку нескольких идентификаторов агентов.

## Обзор

Класс решает проблему сохранения контекста диалога с LLM между сессиями, позволяя каждому агенту иметь собственную историю общения. Автоматически управляет файлами истории, поддерживает переключение между моделями и идентификаторами агентов. Используется в сценариях чат-ботов, виртуальных ассистентов и любых приложений, требующих долговременной памяти диалога с LLM.

## Иерархия и интерфейсы

- **Родительский класс:** `object` (неявно)
- **Реализуемые интерфейсы:** отсутствуют (Python не имеет строгих интерфейсов)
- **Особенности:** класс не предназначен для наследования (нет виртуальных методов)

## Конструкторы

### `__init__(self, api_key: str, base_url: str = "https://api.openai.com/v1", model: str = "gpt-3.5-turbo", temperature: float = 0.7, max_tokens: int = 500, timeout: float = 30.0, verbose: bool = False, agent_id: Optional[str] = None, history_dir: str = "agent_history")`

**Инициализация:**

1. Сохраняет все параметры в соответствующие поля
2. Генерирует `agent_id` (если не передан) из первых 8 символов UUID
3. Создаёт директорию для истории (`history_dir`), если она не существует
4. Формирует имя файла истории: `history_{agent_id}_{model}.json` (небезопасные символы заменяются на `_`)
5. Настраивает логирование при `verbose=True`
6. Автоматически загружает существующую историю из файла (если файл найден)

**Побочные эффекты:** создание директории на диске, возможная загрузка истории из файла.

## Публичные методы

### `reset_conversation(self) -> None`

Сбрасывает текущую историю диалога и сохраняет пустой массив в файл.

**Параметры:** нет  
**Возвращает:** `None`  
**Исключения:** `OSError` (при проблемах с записью файла)  
**Побочные эффекты:** изменяет `conversation_history` и перезаписывает файл истории

---

### `send_message(self, user_message: str) -> str` (асинхронный)

Отправляет сообщение пользователя в LLM и возвращает ответ с автоматическим сохранением истории.

**Параметры:**
- `user_message: str` — текст сообщения пользователя

**Возвращает:** `str` — ответ ассистента (LLM)

**Исключения:**
- `Exception` — при таймауте (`TimeoutException`)
- `Exception` — при HTTP ошибке (статус не 2xx)
- `Exception` — при сетевых ошибках (`RequestError`)
- `Exception` — при некорректном формате ответа API

**Побочные эффекты:**
- Добавляет сообщение пользователя в `conversation_history`
- Добавляет ответ ассистента в `conversation_history`
- Автоматически сохраняет историю в файл
- При `verbose=True` логирует использование токенов

**Изменяет состояние объекта:** да (расширяет `conversation_history`)

**Алгоритм (пошагово):**
1. Добавить сообщение пользователя в историю (роль `"user"`)
2. Сформировать HTTP-запрос к `{base_url}/chat/completions`
3. Отправить асинхронный POST-запрос с таймаутом
4. Обработать возможные исключения (таймаут, HTTP, сеть)
5. Извлечь ответ из JSON-ответа API (`choices[0].message.content`)
6. Добавить ответ ассистента в историю (роль `"assistant"`)
7. Сохранить обновлённую историю в файл
8. Вернуть текст ответа

---

### `save_history(self, filepath: str) -> None`

Сохраняет текущую историю диалога в JSON-файл.

**Параметры:**
- `filepath: str` — путь для сохранения (полный или относительный)

**Возвращает:** `None`  
**Исключения:** `OSError`, `json.JSONEncodeError`  
**Побочные эффекты:** создаёт/перезаписывает файл на диске

---

### `load_history(self, filepath: str) -> None`

Загружает историю диалога из JSON-файла и синхронизирует с дефолтным файлом агента.

**Параметры:**
- `filepath: str` — путь к файлу для загрузки

**Возвращает:** `None`  
**Побочные эффекты:**
- Заменяет текущую `conversation_history` загруженными данными
- Автоматически вызывает `_save_to_default_file()` для синхронизации

**Примечание:** если файл не существует, история сбрасывается в пустой список и логируется предупреждение.

---

### `get_agent_info(self) -> Dict`

Возвращает словарь с метаинформацией об агенте.

**Параметры:** нет  
**Возвращает:** `Dict` с ключами:
- `"agent_id"` — идентификатор агента
- `"model"` — используемая модель
- `"history_file_path"` — путь к файлу истории
- `"history_length"` — количество сообщений в истории

---

### `change_agent_id(self, new_agent_id: str) -> None`

Изменяет идентификатор агента и переименовывает соответствующий файл истории.

**Параметры:**
- `new_agent_id: str` — новый идентификатор (не может быть пустым)

**Возвращает:** `None`  
**Исключения:** `ValueError` — если `new_agent_id` пустой или состоит из пробелов  
**Побочные эффекты:**
- Обновляет `self.agent_id`
- Переименовывает файл истории на диске (если старое имя отличается)
- Обновляет `self._history_file_path`

---

### `list_all_agents(history_dir: str = "agent_history") -> List[Dict]` (статический)

Сканирует директорию истории и возвращает информацию обо всех обнаруженных агентах.

**Параметры:**
- `history_dir: str` — директория для поиска (по умолчанию `"agent_history"`)

**Возвращает:** список словарей, каждый содержит:
- `"filename"` — имя файла
- `"agent_id"` — извлечённый идентификатор
- `"model"` — извлечённая модель
- `"history_file_path"` — полный путь
- `"history_length"` — количество сообщений в истории

**Алгоритм извлечения agent_id и model:**
- Формат файла: `history_{agent_id}_{model}.json`
- Имя очищается от префикса `history_` и суффикса `.json`
- Разделяется по символу `_`; первая часть — `agent_id`, остальные склеиваются в `model`

## Внутренние/приватные методы

### `_sanitize_filename(self, filename: str) -> str`

Заменяет символы, небезопасные для имён файлов в Windows/Linux (`< > : " / \ | ? *`), на подчёркивания.

**Используется для:** предотвращения ошибок при создании файлов с именами, содержащими спецсимволы из `model` или `agent_id`.

---

### `_log(self, msg: str) -> None`

Выводит сообщение в логгер (если `verbose=True`) или печатает с префиксом `[LOG]` в консоль.

---

### `_save_to_default_file(self) -> None`

Приватный метод для автоматического сохранения в файл, определённый при инициализации (`self._history_file_path`). Вызывается после каждого изменения истории.

## Жизненный цикл

1. **Создание** — конструктор `__init__()`, генерируется ID, создаётся директория
2. **Инициализация** — загрузка существующей истории (если есть)
3. **Использование** — многократные вызовы `send_message()` для диалога, возможно переключение ID через `change_agent_id()`
4. **Завершение** — объект может быть уничтожен сборщиком мусора; история уже сохранена на диск после каждого изменения. **Явное закрытие ресурсов не требуется**.

## Пример использования

```python
import asyncio
from agent import Agent

async def main():
    # Создание агента с автоматической генерацией ID
    agent = Agent(
        api_key="your-api-key",
        model="gpt-4",
        verbose=True,
        history_dir="./chat_histories"
    )
    
    print(f"Agent ID: {agent.agent_id}")
    
    # Отправка сообщений (история автоматически сохраняется)
    response1 = await agent.send_message("Привет! Как тебя зовут?")
    print(f"Assistant: {response1}")
    
    response2 = await agent.send_message("Расскажи шутку.")
    print(f"Assistant: {response2}")
    
    # Получение информации об агенте
    info = agent.get_agent_info()
    print(f"History length: {info['history_length']}")
    
    # Сброс истории
    agent.reset_conversation()
    
    # Смена ID агента
    agent.change_agent_id("new_chat_session_001")
    
    # Список всех агентов
    all_agents = Agent.list_all_agents("./chat_histories")
    for a in all_agents:
        print(f"Found agent {a['agent_id']} with {a['history_length']} messages")

# Запуск асинхронной функции
asyncio.run(main())


Примечания по потокам (thread safety)
Класс НЕ потокобезопасен:

Метод send_message() асинхронный, но при конкурентном вызове из нескольких корутин может произойти состояние гонки при записи в conversation_history и файл.

Рекомендуется использовать один экземпляр Agent на корутину или реализовать внешнюю блокировку (asyncio.Lock).

Операции с файлами (save_history, _save_to_default_file) не защищены блокировками.

Связанные классы
